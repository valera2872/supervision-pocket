# Supervision Pocket — alpha 0.7.0

Android-приложение для психологов, которые фиксируют сложные эпизоды после консультаций и готовят конкретные вопросы к супервизии.

## Что реализовано

- понятный онбординг с объяснением назначения приложения;
- полностью локальный запуск без регистрации;
- правила обезличивания клиентских данных;
- PIN 4–6 цифр; в защищённом хранилище находятся только salt и hash;
- блокировка при уходе приложения в background;
- обезличенные карточки клиентов;
- AES-256-GCM-шифрование локального хранилища;
- голосовой ввод через системную службу распознавания речи;
- приложение не сохраняет аудиофайлы;
- последовательная фиксация события, понимания, чувств, импульса, реакции и затруднения;
- автоматическое сохранение черновика;
- сериализация операций записи для защиты от конфликтов автосохранения;
- профессиональная хронология по случаю;
- подготовка вопроса к супервизору;
- передача запроса через системное меню Android или копирование текста;
- архивирование завершённых случаев;
- понятный экран вместо технического красного экрана при ошибке интерфейса.

## Конфиденциальность голосового ввода

Распознавание выполняет системная служба телефона. В зависимости от устройства обработка может происходить локально или сервисом распознавания. Пользователь получает предупреждение не называть имена, адреса, школы и другие идентифицирующие сведения. Само приложение хранит только полученный текст и не сохраняет аудиозапись.

## Запуск

1. Установить актуальный стабильный Flutter SDK и Android SDK.
2. Один раз дополнить сгенерированные platform-файлы:
   `flutter create --platforms=android --org com.supervisionpocket --project-name supervision_pocket .`
3. Выполнить `flutter pub get`.
4. Проверить `dart format`, `flutter analyze` и `flutter test`.
5. Собрать: `flutter build apk --debug`.

## GitHub Actions

Workflow `.github/workflows/flutter-ci.yml` при каждом push в `main`/`develop` и при каждом pull request:

- устанавливает stable Flutter и Java 17;
- добавляет стандартные сгенерированные Android-файлы;
- проверяет форматирование и анализатор;
- запускает тесты;
- собирает debug APK;
- сохраняет APK как artifact запуска workflow на 14 дней.

## Подпись альфа-сборок

Debug APK использует тестовый ключ, сохраняемый в закрытом кэше GitHub Actions. Это позволяет устанавливать новые альфа-сборки поверх предыдущих сборок с той же подписью без потери локальных данных. Перед публикацией в Google Play потребуется отдельный закрытый release/upload key через GitHub Secrets.
