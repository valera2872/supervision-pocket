# Supervision Pocket — alpha 0.2.0

Android-альфа безопасного помощника для профессиональной рефлексии между консультацией и живой супервизией.

## Что реализовано

- полностью локальный запуск без регистрации;
- три шага онбординга;
- явное подтверждение правил обезличивания;
- PIN 4–6 цифр, в защищённом хранилище находятся только salt и hash;
- блокировка при уходе приложения в background;
- главный экран «Сегодня»;
- каркас разделов «Случаи» и «Супервизия»;
- тёплая профессиональная дизайн-система.

## Новое в 0.2.0

- обезличенные карточки случаев;
- AES-256-GCM-шифрование локального хранилища;
- разделение наблюдаемого факта, гипотезы, реакции, импульса и действия;
- автоматическое сохранение черновика;
- профессиональная хронология по случаю;
- подготовленные вопросы к живому супервизору;
- копирование структурированного супервизионного запроса;
- архивирование завершённых случаев.

## Что появится дальше

- голосовой ввод;
- биометрическая разблокировка;
- защищённый PDF-экспорт;
- выводы после супервизии;
- карта повторяющихся профессиональных трудностей.

## Запуск

1. Установить актуальный стабильный Flutter SDK и Android SDK.
2. Один раз дополнить сгенерированные platform-файлы:
   `flutter create --platforms=android --org com.supervisionpocket --project-name supervision_pocket .`
3. Выполнить `flutter pub get`.
4. Проверить `dart format`, `flutter analyze` и `flutter test`.
5. Собрать: `flutter build apk --debug`.

## GitHub Actions

Workflow `.github/workflows/flutter-ci.yml` при каждом push/PR:

- устанавливает stable Flutter и Java 17;
- добавляет стандартные сгенерированные Android-файлы;
- проверяет форматирование и анализатор;
- запускает тесты;
- собирает debug APK;
- сохраняет APK как artifact запуска workflow на 14 дней.

Версии зависимостей перед первой компиляцией должны быть согласованы с установленной версией Flutter.

## Подпись альфа-сборок

Debug APK использует тестовый ключ, сохраняемый в закрытом кэше GitHub Actions. Это позволяет устанавливать новые активные альфа-сборки поверх предыдущих без потери локальных данных и не публиковать ключ в репозитории. Перед выпуском в Google Play будет создан отдельный закрытый release/upload key через GitHub Secrets.
